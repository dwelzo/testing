## multiple test structure in Jest

This is super important when you go beyond 2-3 small tests and start testing multiple functions or modules (like add, subtract, multiply, divide, etc.).


```
import { add, subtract, multiply, divide } from '../app/math';

describe('Math Operations', () => {
  //  ADD
  describe('add()', () => {
    test('adds two positive numbers', () => {
      expect(add(2, 3)).toBe(5);
    });

    test('adds negative numbers', () => {
      expect(add(-2, -3)).toBe(-5);
    });
  });

  //  SUBTRACT
  describe('subtract()', () => {
    test('subtracts smaller from larger', () => {
      expect(subtract(5, 3)).toBe(2);
    });

    test('returns negative when first < second', () => {
      expect(subtract(3, 5)).toBe(-2);
    });
  });

  //  MULTIPLY
  describe('multiply()', () => {
    test('multiplies correctly', () => {
      expect(multiply(2, 4)).toBe(8);
    });

    test('handles zero correctly', () => {
      expect(multiply(0, 5)).toBe(0);
    });
  });

  //  DIVIDE
  describe('divide()', () => {
    test('divides numbers correctly', () => {
      expect(divide(10, 2)).toBe(5);
    });

    test('throws error when dividing by zero', () => {
      expect(() => divide(10, 0)).toThrow('Cannot divide by zero');
    });
  });
});

```



```
| Level               | Purpose                                       | Example                                  |
| ------------------- | --------------------------------------------- | ---------------------------------------- |
| `describe()`        | Groups related tests (like a class or module) | `describe('Math Operations')`            |
| Nested `describe()` | Group smaller parts (like a single function)  | `describe('add()')`                      |
| `test()` or `it()`  | Defines a single test                         | `test('adds two positive numbers', ...)` |
| `expect()`          | Performs the check                            | `expect(add(2,3)).toBe(5)`               |
```
